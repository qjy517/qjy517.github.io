数据结构笔记

2025-05-12

第一章 数据结构绪论

基本概念和术语

数据

就是可描述客观事物的符号集合，要满足两个条件

  1. 能够输入进电脑
  2. 能够被电脑处理

数据元素

就是组成数据的，通常被当做整体处理，被称为记录

数据项

一个数据元素可有若干个数据项组成，可以理解为数据元素为一个结构，那么数据项就是结构中的成员

数据对象

具有相同性质或特征的数据元素的集合，比如人为一个数据对象，其中小明，小白为数据元素，因为他们都有名字，生日，年龄等数据项

数据结构

所谓结构就是数据元素之间的一种或多种关系，元素之间可能平行也可能嵌套等

抽象数据结构

定义

就是一个数学模型和定义在该模型的一系列操作，比如整型，在各大电子产品都有整数类型，且有对应加减操作，尽管其操作方式不同但其数学特性相同，抽象其数学特性和操作就是一个整形的抽象数据结构 再比如线性表，一切有序的数据元素的集合及其操作都可以抽象为线性表

抽象数据类型标准格式

ADT 抽象数据类型名

Data 数据元素之间的逻辑关系的定义

Operation 操作1 初始条件 操作结果描述 操作2 ………..

endADT

算法效率的度量

分为空间上的和时间上的，没有特殊说明就为时间上的

时间复杂度方法：大O法 步骤：

  1. 用常数1代替运行时间中的加法项
  2. 在改后的运行次数函数中，保留最高阶
  3. 最高阶系数化为1

运行时间和运行次数函数就是看for循环部分，语句执行的次数与n的关系，这也是最坏情况

第三章 线性表

定义

就是数据元素的顺序是固定的，这里的顺序并不是特指空间上存储的连续。就是除第一个和最后一个元素外，每个元素都有且只有一个前驱元素和后继元素，注意其中每个数据元素又可以有许多数据项，此时数据元素看做结构体

线性表的抽象数据类型

DATA 线性表

Data 在a1~an的一系列数据元素中，就是除第一个和最后一个元素外，每个元素都有且只有一个前驱元素和后继元素，数据元素之间是一对一关系。

operation 本地链接

线性表的顺序存储结构

是空间上的顺序存储，典型代表为数组

数组长度与线性表长度

线性表长度是有意义的数据元素的长度，而数组长度是分配的内存总长度

地址的计算

对于连续的数据元素a1~an loc(ai+1)=loc(ai)+c loc(ai)=loc(a1)+(i-1)*c，通过此式子，每个数据元素都能用一个式子算出来，为O(1),具有这种存储特点的为随机存储

  2. 线性表长度大于数组长度，抛出异常
  3. 将第i个数据元素后面的所有元素后退一位
  4. 插入元素
  5. 线性表长度加一

线性表顺序存储结构总结

优点：可以快速取出相应的元素，只需知道其下标 缺点：不能够动态管理线性表的长度，容易造成数据溢出，或存储浪费，在插入与删除操作需要移动大量元素，为 O(n);

  2. 否则查找成功

插入和删除操作

插入和删除:改变指针域指向就行，需要malloc和free函数

插入第i个节点算法思路：

  1. 声明指针p指向头结点，初始化j
  2. 当j<i时，就遍历链表，p指向下一个节点，j累加
  3. 若p在链表末尾指向空，则未查找到相应节点
  4. 否则查找到相应的节点，生成一个空节点
  5. 将数据e赋给其数据域
  6. 链接前驱结点与后继节点，标准插入语句：s->next=p->next,p->next=s;

void LinkInsert(LinkList *L,int e,int i )//对于形参采用的是LinkList *这是便于头插法的插入，因为头指针会发生改变
{
    int j=1;
    LinkList p,s;
    p=*L; //这就是头结点，把&head传入，解后就是head节点
   while (j<i&&p) //用while循环更好，因为for循环不能够判断p是否为空，如果i过大，单一的j<i就会一直循环直到j>i，                                  造成时间复杂度
   {
       p=p->next;     //有了头结点p就指向了第i-1个节点  
       j++;
   }
    if (!p||j>i)//j>i指i可能为负数
    {
        return error;
    }
    s=(LinkList)malloc(sizeof(Node));
    s->data=e;
    s->next=p->next;//这里前面已将判定了p!=NULL,所以可以对p进行操作
    p-next=s;
    return ok;
}

删除节点算法：

  1. 声明指针p，初始化j
  2. j<i就循环，p更新，j累加
  3. 当p指向链表末尾为空，表示没有查找到相应的节点
  4. 否则，就删除节点，即p-next为待删除的节点，将p->next=p->next->next

void LinkDelete(LinkList *L,int i)
{
    LinkList p=*L;
    int j;
    LinkList q;
    while ((p->next)&&j<i)
    {
        p=p->next;
        j++

    if ((p->next)||j>i)
    {
        return error;
    }
    q=p->next;
}

这里应用了p-next而不是p，效果是一样的，正确退出循环时，p仍指向i-1个节点，不同的于p的是，对于没找到节点到末尾的情况，p-next为NULL，则p指向最后一个节点，后续还能够对该节点操作，但p=NULL，后续就不能对该节点操作

单链表的整表创建

不同于声明数组，单链表的创建时一个空表逐渐插入各个节点的动态过程 创建算法：

  1. 声明一个指针，计数器变量i
  2. 创建一个头结点，并使其指针域为NULL即带有头结点的空链表
  3. 循环：
    1. 生成一新节点赋给p
    2. 给p的指针域和数据域赋值
    3. 将p链接到上一节点

void CreatLinkList(LinkList *L,int n)
{
    LinkList p;
    int i;
    srand(time(0));
    *L=(LinkList)malloc(sizeof(Node));
    (*L)->next=NULL;
    p=(LinkList)malloc(sizeof(Node));
    for (int i=1;i<=n;i++)
    {
        p=(LinkList)malloc(sizeof(Node));
        p->data=rand()%100+1;

}

void LinkListDelete(LinkList *L)
{

    }
}

循环链表

作用：希望能从一个节点出发访问全部元素

单链表循环

就是将末尾指针指向头结点而非NULL； 与单链表的主要差异：就是循环条件改变，原来是p->next=NULL,现在是p->next等于头结点则循环结束

但此时对于末尾节点的访问仍为O(n),怎么能变为O(1),就是将头指针改为尾指针

这样指针头指针就指向第一个结点，然后next就是指向下一个节点即头结点

改为尾指针还能够将两个循环链表连接起来合成一个表：

  1. 只需要第一个链表的末尾节点不在指向头结点而指向第二个链表的第一个节点
  2. 将第二个链表的最后一个节点指向第一个链表的头结点
  3. 释放第二个链表的头结点

线性表总结

  1. 线性表按存储结构划分为顺序和链式结构，前者适用于多读取，少插入，后者适用于数据量不定，插入多

对与要用LinkStack*s形参的解释：

如果按值传递（即函数参数为 LinkStack S ，而不是 LinkStack *S ），函数内部对 S 的操作是对传入实参的副本进行的。

  • 无法修改原链栈：函数内修改 S 的 top 指针等成员变量，只是改变了这个副本的值，函数结束后，副本被销毁，原链栈的 top 等属性并不会发生改变。例如在入栈操作中，即使在函数内调整了副本的 top 指针指向新节点，但原链栈的 top 还是指向原来位置，新元素并没有真正入栈。
  • 失去信息同步：对于记录元素个数的 count 成员也是同理，函数内修改副本的 count ，原链栈的 count 不会变化，导致链栈实际元素数量和记录数量不一致，破坏了链栈数据结构的完整性和准确性 。所以按值传递不能实现对原链栈的有效操作。

栈的应用

1.递归

存储某些数据，并在后面又以存储的逆序恢复这些数据，符合栈的先进后出，后进先出的特点

简单说就是在前进的过程中(递归调用的过程)，函数的局部变量，参数值以及返回地址都被压入栈中，在退回阶段，位于栈顶的局部变量，参数值，及返回地址被弹出，用于返回调用代码的其余部分

2. 四则运算表达式

计算机对与四则运算就是一个压栈出栈的过程，首先是将四则运算转化为后缀表达式，再将后缀表达式压入栈中

转化规则：

从左到右遍历表达式中的数字和符号，若是数字就输出，若是符号，就判断与栈顶符号的优先级，是右括号或者优先级不高于栈顶符号(乘除优先加减),就依次弹出栈顶符号并输出，并将当前符号进栈，一直到最终输出后缀表达式为止

队列

定义

类比排队办业务

队列是一种先进先出的线性表，简称FIFO，他只允许一端插入，一端删除，允许插入的一端是队尾，允许删除的一端是队头 队列的抽象数据类型见源码

队列的顺序存储结构及实现

与栈类比，队列同样有顺序存储结构和链式存储结构

对与顺序存储结构，即数组，为了弥补在头部元素出走，而移动后面全部元素的情况(O(n)),我们引入了两个类似指针的变量，来指向队头和队尾的下一个元素，这是为了当front=tail时，队列为空，而不是只剩下一个元素

但同时又引出了新的问题：假溢出（看书）

这样为了充分利用空间，就要用到循环队列，即tail不在指向数组之外，而是指向头部

那么在循环链表又会出现一种情况就是，front=tail同样为队列满，如何区分队列空与满呢？

  1. 用标志变量

  2. 设定队列为满新的规则：

    当队列中只有一个空元素时，队列是满的

如何判断队列只有一个空元素？

假设数组最大长度为Max，公式：(tail+1)%Max==front,这就是列表满了

通用的计算队列长度公式：Length=(tail-front+Max)%Max

二叉树

特点

  1. 每个节点只有两棵树
  2. 左右节点不能换
  3. 只有一个节点时，也要区分是左还是右

分类：

  1. 斜树：只有左节点或右节点
  2. 满二叉树：每一层都是满的
  3. 完全二叉树：每个节点顺序与满二叉树相同，即最后一层之前的每一层都是满的，对与最后一层，节点必须按顺序出现，不能一个节点有右无左

性质：

  1. 第i层最多有2^(i-1)个节点
  2. 深度为k的是树最多有2^k-1个节点
  3. 对与任意一个二叉树，它的终端节点（n0）和度为2的节点（n2）的关系：n0=n2+1；
  4. 有n个节点的完全二叉树，它的层数为[log下2上（n+1）]+1
  5. 对与n个节点的完全二叉树，（深度为[log下2上（n+1）]+1]，的节点按层序编号，对与任意几点有
    • 如果i=1，则·节点是根几点，无双亲；如果i>1，则双亲是[i/2]
    • 如果2i>n,则节点无左孩子（即节点i为叶子节点），否则其左孩子时节点2i；
    • 如果2i+1>n则节点无右孩子，否则右孩子是2i+1

中序遍历
